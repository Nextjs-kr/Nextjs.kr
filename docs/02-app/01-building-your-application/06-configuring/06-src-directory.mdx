프로젝트 루트에 특별한 Next.js `app` 또는 `pages` 디렉터리들을 두는 대신, Next.js는 `src` 디렉터리 아래에 애플리케이션 코드를 배치하는 일반적인 패턴도 지원합니다.

대부분 프로젝트의 루트에 있는 프로젝트 구성 파일에서 애플리케이션 코드가 분리됩니다. 일부 개인과 팀이 선호하는 방식입니다.

`src` 디렉터리를 사용하려면 `app` 라우터 폴더 또는 `pages` 라우터 폴더를 각각 `src/app` 또는 `src/pages`로 옮기세요.

> **알아두면 좋은 정보**
>
> - `/public` 디렉터리는 프로젝트의 루트에 있어야 합니다.
> - `package.json`, `next.config.js`, `tsconfig.json`와 같은 구성 파일은 프로젝트의 루트에 남아있어야 합니다.
> - `.env.*` 파일은 프로젝트의 루트에 있어야 합니다.
> - 루트 디렉터리에 `app` 또는 `pages`가 있는 경우 `src/app` 또는 `src/pages`는 무시됩니다.
> - `src`를 사용하는 경우 `/components` 또는 `/lib`과 같은 다른 애플리케이션 폴더들도 이동해야 할 것입니다.
